First entry. Today I set up this diary.

The stack is [Astro](https://astro.build/) using the official blog template, deployed
on [Netlify](https://www.netlify.com/). The workflow I'm going for is deliberately
low-friction:

1. Write a Markdown file in `src/content/blog/` — one per day.
2. Commit and push to GitHub.
3. Netlify rebuilds and publishes automatically.

That's it. No CMS, no database, just text files in git.

## Writing a new entry

Each post is a Markdown file with a little frontmatter block at the top:

```markdown
---
title: 'A short title for the day'
description: 'One line summary (shows up in the RSS feed and previews).'
pubDate: 'Jul 02 2026'
---

Whatever I did today...
```

`title`, `description`, and `pubDate` are required. `heroImage` and `updatedDate`
are optional. Posts are sorted newest-first automatically.
